## Added
- Added `Include prerelease updates` in settings, below `Check program updates on start`.
- Added `Enable colored logs` in `Settings > Core`, enabled by default.
- Added more detail to the `Connection` tab, including process, protocol, and traffic columns.

## Changed
- Centralized the app version into the root `VERSION` file.
- Updated the start/stop button behavior so it switches to `Stop` immediately while startup is in progress.
- Kept `build/` in the project root while preserving the cleaner deployment layout.
- Improved the initial width behavior of the `Traffic` column so it no longer starts too narrow.

## Fixed
- Fixed ANSI-colored logs not rendering correctly in the `Log` tab.
- Fixed prerelease update checks to follow GitHub’s official prerelease flag.
- Fixed the `Connection` tab so active connections are shown again.
- Fixed connection tracking so the `Connection` tab keeps updating even when traffic refresh is disabled.
- Fixed empty process names in the `Connection` tab by enabling runtime process lookup and improving process-name fallback handling.
- Removed the warning dialog shown when pressing `Start` repeatedly while a profile is already starting.

## Cleanup
- Removed unused assets and stale references that were no longer part of the build/runtime.
